## 🔍 根本原因确认

### 发现 1: 消息内容是压缩格式

通过检查消息数据库，发现：

```sql
-- 消息表结构
CREATE TABLE Chat_{hash} (
    CreateTime INTEGER,
    Des INTEGER,
    ImgStatus INTEGER,
    MesLocalID INTEGER PRIMARY KEY,
    Message TEXT,        -- 消息内容字段
    MesSvrID INTEGER,
    Status INTEGER,
    TableVer INTEGER,
    Type INTEGER
);
```

**关键发现**：`Message` 字段存储的是**压缩的二进制数据**，不是纯文本 XML。

示例（十六进制）：
```
28B52FFD6105A704050B00938D2CB188211A07E6442771D30450129813A51F...
```

这是 **protobuf 或其他二进制格式**，需要先解压缩才能解析。

### 发现 2: WechatExporter 的解析逻辑

查看 WechatExporter 源码：

```cpp
// MessageParser.cpp
bool MessageParser::parseAppMsgAttachment(...) {
    // 直接解析 XML
    xmlParser.parseNodeValue("/msg/appmsg/title", title);
    xmlParser.parseNodeValue("/msg/appmsg/appattach/fileext", attachFileExtName);
    ...
}
```

**问题**：WechatExporter 假设 `Message` 字段是 XML 文本，但实际上是压缩的二进制数据。

### 发现 3: 微信版本差异

- **旧版本微信**（< 8.0）：消息存储为纯文本 XML
- **新版本微信**（≥ 8.0）：消息存储为压缩的 protobuf 格式

**您的环境**：
- 微信版本：8.0.9
- iOS 版本：15.0
- WechatExporter 版本：v1.8.0.10（可能不支持新格式）

### 方案 1: 修改 WechatExporter 支持压缩消息（推荐）

#### 1.1 添加消息解压缩功能

在 `MessageParser.cpp` 中添加解压缩逻辑：

```cpp
// 新增：消息解压缩函数
std::string MessageParser::decompressMessage(const std::string& compressed) const
{
    // 检测压缩格式
    if (compressed.empty() || compressed.size() < 2) {
        return compressed;  // 未压缩
    }
    
    unsigned char header = (unsigned char)compressed[0];
    
    // 检查是否是 protobuf 格式（通常以 0x28 开头）
    if (header == 0x28) {
        // 使用 protobuf 解析
        // TODO: 需要微信的 protobuf 定义文件
        return decompressProtobuf(compressed);
    }
    
    // 检查是否是 zlib/gzip 压缩
    if (header == 0x78 || header == 0x1f) {
        return decompressZlib(compressed);
    }
    
    // 未压缩，直接返回
    return compressed;
}

// Zlib 解压缩
std::string MessageParser::decompressZlib(const std::string& compressed) const
{
    z_stream stream;
    memset(&stream, 0, sizeof(stream));
    
    if (inflateInit(&stream) != Z_OK) {
        return "";
    }
    
    stream.avail_in = compressed.size();
    stream.next_in = (Bytef*)compressed.data();
    
    std::string decompressed;
    char buffer[4096];
    
    do {
        stream.avail_out = sizeof(buffer);
        stream.next_out = (Bytef*)buffer;
        
        int ret = inflate(&stream, Z_NO_FLUSH);
        if (ret == Z_STREAM_ERROR) {
            inflateEnd(&stream);
            return "";
        }
        
        decompressed.append(buffer, sizeof(buffer) - stream.avail_out);
    } while (stream.avail_out == 0);
    
    inflateEnd(&stream);
    return decompressed;
}
```

### 方案 2: 使用 Python 脚本手动提取附件（快速方案）

编写 Python 脚本直接从 iTunes 备份提取文档附件。

#### 2.1 脚本实现

```python
#!/usr/bin/env python3
"""
从 iTunes 备份中提取微信文档附件
"""

import sqlite3
import os
import shutil
from pathlib import Path

# 配置
BACKUP_PATH = os.path.expanduser("~/Library/Application Support/MobileSync/Backup/00008140-000139E62247001C")
OUTPUT_PATH = os.path.expanduser("~/Documents/WechatAttachments")
MANIFEST_DB = os.path.join(BACKUP_PATH, "Manifest.db")

def extract_attachments():
    """提取所有文档附件"""
    
    # 连接 Manifest.db
    conn = sqlite3.connect(MANIFEST_DB)
    cursor = conn.cursor()
    
    # 查询所有文档附件
    query = """
    SELECT fileID, relativePath 
    FROM Files 
    WHERE relativePath LIKE '%OpenData%' 
      AND (relativePath LIKE '%.pdf' 
        OR relativePath LIKE '%.doc' 
        OR relativePath LIKE '%.docx' 
        OR relativePath LIKE '%.xls' 
        OR relativePath LIKE '%.xlsx'
        OR relativePath LIKE '%.ppt'
        OR relativePath LIKE '%.pptx'
        OR relativePath LIKE '%.zip'
        OR relativePath LIKE '%.rar'
        OR relativePath LIKE '%.7z')
    """
    
    cursor.execute(query)
    files = cursor.fetchall()
    
    print(f"找到 {len(files)} 个文档附件")
    
    # 创建输出目录
    os.makedirs(OUTPUT_PATH, exist_ok=True)
    
    # 提取文件
    extracted = 0
    for file_id, relative_path in files:
        # 构建源文件路径
        # iTunes 备份使用 fileID 的前两个字符作为子目录
        src_path = os.path.join(BACKUP_PATH, file_id[:2], file_id)
        
        if not os.path.exists(src_path):
            print(f"⚠️  文件不存在: {file_id}")
            continue
        
        # 提取文件名和扩展名
        filename = os.path.basename(relative_path)
        
        # 提取会话 hash（用于分类）
        parts = relative_path.split('/')
        if len(parts) >= 4:
            session_hash = parts[3]  # OpenData/{hash}/{msgId}.ext
        else:
            session_hash = "unknown"
        
        # 创建会话目录
        session_dir = os.path.join(OUTPUT_PATH, session_hash)
        os.makedirs(session_dir, exist_ok=True)
        
        # 复制文件
        dest_path = os.path.join(session_dir, filename)
        try:
            shutil.copy2(src_path, dest_path)
            extracted += 1
            if extracted % 100 == 0:
                print(f"已提取 {extracted}/{len(files)} 个文件...")
        except Exception as e:
            print(f"❌ 复制失败: {filename} - {e}")
    
    conn.close()
    
    print(f"\n✅ 完成！共提取 {extracted} 个文档附件")
    print(f"📁 输出目录: {OUTPUT_PATH}")
    
    # 统计各类型文件数量
    print("\n📊 文件类型统计:")
    extensions = {}
    for root, dirs, files in os.walk(OUTPUT_PATH):
        for file in files:
            ext = os.path.splitext(file)[1].lower()
            extensions[ext] = extensions.get(ext, 0) + 1
    
    for ext, count in sorted(extensions.items(), key=lambda x: x[1], reverse=True):
        print(f"  {ext}: {count} 个")

if __name__ == "__main__":
    extract_attachments()
```

## 🔬 技术细节

### 消息压缩格式分析

微信 8.0+ 使用的消息格式：

1. **Protobuf 编码**
   - 字段定义需要逆向工程
   - 可能使用 Google Protobuf 或微信自定义格式

2. **Zlib 压缩**
   - 标准 zlib/gzip 压缩
   - 可以使用标准库解压

3. **混合格式**
   - 部分消息压缩，部分不压缩
   - 需要检测并分别处理

### WechatExporter 兼容性

| 微信版本 | 消息格式 | WechatExporter 支持 |
|---------|---------|-------------------|
| < 7.0 | 纯文本 XML | ✅ 完全支持 |
| 7.0 - 7.9 | 纯文本 XML | ✅ 完全支持 |
| 8.0+ | 压缩 Protobuf | ❌ 不支持 |
