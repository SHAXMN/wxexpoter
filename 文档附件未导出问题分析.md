## 🔎 原因分析

### 1. 代码层面分析

WechatExporter 的代码是正确的：

```cpp
// MessageParser.cpp - parseAppMsgAttachment 函数
bool MessageParser::parseAppMsgAttachment(const WXAPPMSG& appMsg, ...) {
    // 解析文件扩展名
    xmlParser.parseNodeValue("/msg/appmsg/appattach/fileext", attachFileExtName);
    
    // 构建文件路径
    std::string attachFileName = m_userBase + "/OpenData/" + session.getHash() + "/" + appMsg.msg->msgId;
    if (!attachFileExtName.empty()) {
        attachFileName += "." + attachFileExtName;
    }
    
    // 调用 parseFile 复制文件
    return parseFile(..., attachFileName, ...);
}
```

代码逻辑：
1. ✅ 从消息 XML 中提取文件扩展名
2. ✅ 构建正确的 OpenData 路径
3. ✅ 调用 copyFile 从 iTunes 备份复制文件

### 2. 可能的原因

#### 原因 A: 消息类型识别问题（最可能）

WechatExporter 可能没有正确识别文档附件消息。可能的情况：

1. **消息类型不是 APPMSGTYPE_ATTACH (6)**
   - 微信可能使用了其他消息类型来表示文档附件
   - 例如：APPMSGTYPE_OPEN (7) 或其他未知类型

2. **消息 XML 结构不同**
   - 新版本微信的消息格式可能有变化
   - `fileext` 字段可能在不同的 XML 路径下

3. **会话过滤**
   - WechatExporter 可能只导出了部分会话
   - 文档附件可能在未导出的会话中

#### 原因 B: 文件路径映射问题

iTunes 备份使用哈希文件名，WechatExporter 需要通过 Manifest.db 映射：

```
实际路径: Documents/.../OpenData/{hash}/{msgId}.pdf
备份文件: {fileID} (40位哈希)
```

如果映射失败，`copyFile` 会找不到文件。

#### 原因 C: 导出选项设置

WechatExporter 可能有导出选项：
- 只导出图片和视频
- 跳过大文件
- 跳过某些消息类型
