## 待完成任务

### 1. 图片解密问题 [PENDING]
**状态**: 暂时搁置，待后续处理

**问题描述**:
- 微信使用 `wxgf` 格式加密本地存储的图片
- 已实现解密工具 `decrypt_wechat_images.py`
- 解密后的图片在 macOS 预览中仍然无法打开

**已完成工作**:
- ✅ 识别 wxgf 加密格式（文件头：0x77 0x78 0x67 0x66）
- ✅ 实现 XOR 解密算法（密钥：0xFE）
- ✅ 自动检测偏移量（通常在 0x7F-0x84 之间）
- ✅ `file` 命令可以识别解密后的文件为 JPEG
- ❌ macOS 预览应用无法打开解密后的图片

**可能原因**:
1. JPEG 文件结构不完整（缺少 EOI 标记 0xFF 0xD9）
2. 解密偏移量不准确，导致文件头部有额外数据
3. 微信可能使用了更复杂的加密方式（不仅仅是简单 XOR）
4. 图片元数据损坏

**下一步行动**（待后续处理）:
- [ ] 使用 `xxd` 检查解密后文件的完整结构
- [ ] 验证 JPEG 文件是否有正确的 EOI 标记
- [ ] 尝试使用 ImageMagick 或其他工具修复图片
- [ ] 研究微信最新版本的加密方式变化
- [ ] 考虑使用微信官方 API 或第三方工具

**相关文件**:
- `decrypt_wechat_images.py` - 解密工具脚本
- 测试文件位置：`~/Library/Containers/com.tencent.xinWeChat/.../MessageTemp/*/Image/*.pic.dftemp.jpg`

### 图片解密问题备注
**日期**: 2026-04-17

虽然解密工具可以正确识别和解密 wxgf 格式，但解密后的图片在 macOS 预览中无法打开。考虑到：
1. 这不是项目的核心功能
2. 可能需要深入研究微信的加密实现
3. 时间成本较高

决定暂时搁置此问题，优先处理项目的其他功能。后续如果有时间或找到更好的解决方案，再回来处理。

可能的替代方案：
- 使用微信 PC 版导出图片
- 使用第三方微信数据恢复工具
- 直接从 iTunes 备份中提取未加密的图片（如果存在）
