## 🔧 编译步骤

### 方法 1: 使用 Xcode GUI（推荐）

#### 步骤 1: 打开项目

```bash
cd /Users/HuBin/Documents/Quant/WechatExporter
open WechatExporter.xcodeproj
```

#### 步骤 2: 选择 Scheme

在 Xcode 顶部工具栏：
1. 点击 Scheme 下拉菜单（左侧）
2. 选择 **WechatExporter**
3. 选择目标设备：**My Mac**

#### 步骤 3: 配置 Build Settings

1. 点击项目名称（左侧导航栏顶部）
2. 选择 **WechatExporter** target
3. 点击 **Build Settings** 标签
4. 搜索 "Deployment Target"
5. 确保 **macOS Deployment Target** 设置为 **12.0** 或更高

#### 步骤 4: 清理并编译

```
Product → Clean Build Folder (Shift + Cmd + K)
Product → Build (Cmd + B)
```

#### 步骤 5: 查看编译结果

- **成功**: 顶部显示 "Build Succeeded"
- **失败**: 查看错误信息（见故障排除部分）

#### 步骤 6: 运行应用

```
Product → Run (Cmd + R)
```

应用会自动启动。

## 🐛 故障排除

### 问题 1: 编译错误 - "file not found"

**错误信息**：
```
fatal error: 'plist/plist.h' file not found
```

**原因**: 缺少依赖库

**解决方案**：
```bash
# 安装 Homebrew（如果未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装依赖
brew install libplist protobuf
```

然后在 Xcode 中：
1. 选择项目 → Build Settings
2. 搜索 "Header Search Paths"
3. 添加：`/usr/local/include`
4. 搜索 "Library Search Paths"
5. 添加：`/usr/local/lib`

---

### 问题 2: 运行时错误 - "无法打开应用"

**错误信息**：
```
"WechatExporter.app" 已损坏，无法打开
```

**原因**: macOS Gatekeeper 安全限制

**解决方案**：
```bash
# 移除隔离属性
xattr -cr ./build/Build/Products/Release/WechatExporter.app

# 或者在系统偏好设置中允许
# 系统偏好设置 → 安全性与隐私 → 通用 → 仍要打开
```

---

### 问题 3: 附件未导出

**症状**: 日志显示 "Found 0 document attachment(s)"

**可能原因**：
1. iTunes 备份中确实没有附件
2. 文件路径不正确

**诊断步骤**：
```bash
# 检查 iTunes 备份
cd ~/Library/Application\ Support/MobileSync/Backup/00008140-000139E62247001C

# 查询附件数量
sqlite3 Manifest.db "SELECT COUNT(*) FROM Files WHERE relativePath LIKE '%OpenData%' AND (relativePath LIKE '%.pdf' OR relativePath LIKE '%.doc%');"
```

**解决方案**：
- 如果返回 0：备份中确实没有附件
- 如果返回 > 0：检查代码中的路径逻辑

---

### 问题 4: HTML 中无附件链接

**症状**: 聊天记录中显示 "[Link]" 而不是附件链接

**可能原因**：
1. 消息是压缩格式且 OpenData 中找不到文件
2. 文件扩展名不在支持列表中

**诊断步骤**：
```bash
# 检查调试日志（如果是 Debug 版本）
ls ~/Documents/WechatExport_Test/../dbg/msg_49_*.txt

# 查看消息内容
cat ~/Documents/WechatExport_Test/../dbg/msg_49_app_invld_*.txt | head -20
```

**解决方案**：
- 检查 `detectExtensionFromOpenData()` 的扩展名列表
- 添加更多文件类型支持

---

### 问题 5: 点击链接 404

**症状**: 点击附件链接显示 "File not found"

**可能原因**：
1. 文件未复制到 attachments 目录
2. 链接路径不正确

**诊断步骤**：
```bash
# 检查文件是否存在
ls -la ~/Documents/WechatExport_Test/attachments/*/

# 检查 HTML 中的链接
grep -o 'href="../attachments/[^"]*"' ~/Documents/WechatExport_Test/湖滨/*.html | head -5
```

**解决方案**：
- 验证 `copyFile()` 是否成功
- 检查相对路径是否正确

## 🚀 快速测试脚本

```bash
#!/bin/bash
# 快速测试脚本

echo "=== WechatExporter 快速测试 ==="

# 1. 检查编译产物
echo "1. 检查编译产物..."
if [ -d "./build/Build/Products/Release/WechatExporter.app" ]; then
    echo "✅ 编译产物存在"
else
    echo "❌ 编译产物不存在，请先编译"
    exit 1
fi

# 2. 检查 iTunes 备份
echo "2. 检查 iTunes 备份..."
BACKUP_PATH=~/Library/Application\ Support/MobileSync/Backup/00008140-000139E62247001C
if [ -d "$BACKUP_PATH" ]; then
    echo "✅ iTunes 备份存在"
else
    echo "❌ iTunes 备份不存在"
    exit 1
fi

# 3. 运行应用
echo "3. 启动应用..."
open ./build/Build/Products/Release/WechatExporter.app

echo ""
echo "请在应用中完成导出，然后按回车继续..."
read

# 4. 检查导出结果
echo "4. 检查导出结果..."
OUTPUT_DIR=~/Documents/WechatExport_Test

if [ -d "$OUTPUT_DIR/attachments" ]; then
    echo "✅ attachments 目录已创建"
    
    ATTACHMENT_COUNT=$(find "$OUTPUT_DIR/attachments" -type f | wc -l | tr -d ' ')
    echo "   附件数量: $ATTACHMENT_COUNT"
    
    if [ $ATTACHMENT_COUNT -gt 0 ]; then
        echo "✅ 附件已导出"
    else
        echo "⚠️  attachments 目录为空"
    fi
else
    echo "❌ attachments 目录不存在"
fi

# 5. 检查 HTML 链接
echo "5. 检查 HTML 链接..."
HTML_FILES=$(find "$OUTPUT_DIR" -name "*.html" -not -path "*/assets/*")
LINK_COUNT=$(grep -o 'href="../attachments/' $HTML_FILES | wc -l | tr -d ' ')

if [ $LINK_COUNT -gt 0 ]; then
    echo "✅ 找到 $LINK_COUNT 个附件链接"
else
    echo "⚠️  未找到附件链接"
fi

echo ""
echo "=== 测试完成 ==="
```

保存为 `test.sh` 并运行：
```bash
chmod +x test.sh
./test.sh
```
