# WechatExporter GitHub Actions 构建错误修复总结

## ✅ 已完成的修复

### 1. **C++ 标准升级** (最关键)
- **修改**: `gnu++14` → `gnu++17`
- **原因**: protobuf 新版本依赖 abseil 库,abseil 强制要求 C++17+
- **影响**: 解决了 20+ 个编译错误 (std::invoke, std::conjunction, std::apply 等)

### 2. **macOS 部署目标升级**
- **修改**: `10.10` → `10.13`
- **原因**: Xcode 26.3 支持范围是 10.13-26.2.99
- **影响**: 消除部署目标警告

### 3. **添加音频库依赖**
- **opencore-amr**: AMR 音频编解码 (微信语音消息格式之一)
- **lame**: MP3 编解码
- **silk**: 微信专用的语音编解码库 (从源码编译)

### 4. **更新构建配置**
- 添加所有依赖库的头文件搜索路径
- 添加所有依赖库的库文件搜索路径
- 确保 xcodebuild 能找到所有依赖

## 🔍 关键问题解决

### 问题 1: C++17 特性缺失
```
错误: error: "C++ versions less than C++17 are not supported."
错误: no member named 'invoke' in namespace 'std'
错误: no member named 'conjunction' in namespace 'std'
```
**解决**: 升级到 C++17 标准

### 问题 2: silk 库缺失
```
错误: fatal error: 'silk/SKP_Silk_SDK_API.h' file not found

**解决**: 从 https://github.com/collects/silk 克隆并编译

```
错误: 'opencore-amrnb/interf_dec.h' file not found

**解决**: 通过 brew 安装 opencore-amr 和 lame

## 📚 技术细节

### silk 库编译过程
```bash
cd /tmp
git clone https://github.com/collects/silk.git
cd silk
make lib  # 生成 libSKP_SILK_SDK.a
sudo cp interface/*.h /usr/local/include/silk/
sudo cp libSKP_SILK_SDK.a /usr/local/lib/
```

### 依赖库列表
- **jsoncpp**: JSON 解析
- **protobuf**: Protocol Buffers (微信消息格式)
- **abseil**: protobuf 的依赖 (要求 C++17)
- **libplist**: iOS 备份文件解析
- **opencore-amr**: AMR 音频编解码
- **lame**: MP3 编解码
- **silk**: 微信语音编解码 (手动编译)

## ⚠️ 注意事项

1. **C++17 要求**: 不再支持 macOS 10.10-10.12
2. **silk 编译时间**: 首次构建会增加约 1-2 分钟
3. **依赖版本**: 使用 Homebrew 最新稳定版本
