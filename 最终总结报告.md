## 🔧 实现的功能

### 1. 批量附件导出（任务 #14）

**功能**：
- 自动扫描 iTunes 备份中的所有文档附件
- 支持 10 种文件类型：.pdf, .doc, .docx, .xls, .xlsx, .ppt, .pptx, .zip, .rar, .7z
- 按会话 hash 分类保存到 `attachments/` 目录

**实现文件**：
- `ITunesParser.h`: 添加 `AttachmentInfo` 结构和方法声明
- `ITunesParser.cpp`: 实现 `scanDocumentAttachments()` 和 `copyAttachments()`
- `Exporter.cpp`: 集成到导出流程

**代码示例**：
```cpp
// 扫描附件
std::vector<AttachmentInfo> attachments;
if (m_iTunesDb->scanDocumentAttachments(attachments))
{
    // 复制附件
    int copied = m_iTunesDb->copyAttachments(attachments, m_output);
}
```

**功能**：
- 在聊天记录中显示可点击的附件链接

## 🔍 技术细节

### 消息处理流程

```
Type 49 (APPMSG) 消息
    ↓
尝试解析 XML
    ↓
成功？
├─ 是 → parseAppMsgAttachment()
│         ↓
│      提取原始文件名和扩展名
│         ↓
│      如果扩展名缺失 → detectExtensionFromOpenData()
│         ↓
│      获取文件大小
│         ↓
│      复制到 attachments/{sessionHash}/
│         ↓
│      生成 HTML 链接（包含文件大小）
│
└─ 否 → tryParseCompressedAttachment()
          ↓
       detectExtensionFromOpenData()
          ↓
       找到文件？
       ├─ 是 → 获取文件大小
       │       ↓
       │    复制并生成链接
       │
       └─ 否 → 显示 "[Link]"
```

### 关键算法

#### 1. 扩展名检测

```cpp
std::string detectExtensionFromOpenData(const std::string& msgId, const std::string& sessionHash)
{
    const char* extensions[] = {".pdf", ".doc", ".docx", ...};
    
    for (const char* ext : extensions)
    {
        std::string vpath = m_userBase + "/OpenData/" + sessionHash + "/" + msgId + ext;
        
        if (m_iTunesDb.findITunesFile(vpath) != nullptr)
        {
            return ext;
        }
    }
    
    return "";
}
```

#### 2. 文件大小格式化

```cpp
std::string formatFileSize(size_t bytes)
{
    if (bytes >= GB) return "X.XX GB";
    if (bytes >= MB) return "X.XX MB";
    if (bytes >= KB) return "X.XX KB";
    return "X B";
}
```

#### 3. 批量扫描

```cpp
bool scanDocumentAttachments(std::vector<AttachmentInfo>& attachments)
{
    for (const auto* file : m_files)
    {
        if (file->relativePath.find("/OpenData/") != std::string::npos)
        {
            if (isDocumentFile(file->relativePath))
            {
                attachments.push_back(extractInfo(file));
            }
        }
    }
}
```

## 🐛 已知限制

### 1. 文件名是 msgId

**问题**: 压缩消息无法获取原始文件名

**影响**: 下载的文件名是 `123456.pdf` 而不是 `项目报告.pdf`

**解决方案**: 
- 短期：用户可接受
- 长期：逆向 protobuf 结构

### 2. 部分消息无法解析

**问题**: 某些压缩消息格式未知

**影响**: 显示 "[Link]" 而不是附件链接

**解决方案**: 
- 添加更多文件类型支持
- 改进检测算法

### 3. 无附件预览

**问题**: 无法在 HTML 中预览 PDF 等文件

**影响**: 必须下载才能查看

**解决方案**: 
- 未来添加 PDF.js 预览
- 添加图片缩略图
