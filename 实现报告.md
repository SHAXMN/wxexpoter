## 🔧 技术实现

### 1. 数据结构（ITunesParser.h）

```cpp
// Document attachment info for batch extraction
struct AttachmentInfo
{
    std::string fileId;         // iTunes 备份中的文件 ID
    std::string relativePath;   // 相对路径
    std::string sessionHash;    // 会话 hash（用于分类）
    std::string filename;       // 文件名

    AttachmentInfo() {}
};
```

### 2. 扫描附件（ITunesParser.cpp）

```cpp
bool ITunesDb::scanDocumentAttachments(std::vector<AttachmentInfo>& attachments) const
```

**功能**：
- 遍历 iTunes 备份中的所有文件
- 筛选 OpenData 目录下的文档文件
- 支持的扩展名：.pdf, .doc, .docx, .xls, .xlsx, .ppt, .pptx, .zip, .rar, .7z
- 提取会话 hash 和文件名

**实现逻辑**：
1. 遍历 `m_files` 中的所有文件
2. 检查路径是否包含 `/OpenData/`
3. 检查文件扩展名（不区分大小写）
4. 从路径中提取会话 hash：`Documents/.../OpenData/{hash}/{msgId}.ext`
5. 提取文件名

### 3. 复制附件（ITunesParser.cpp）

```cpp
int ITunesDb::copyAttachments(const std::vector<AttachmentInfo>& attachments, const std::string& outputPath) const
```

**功能**：
- 批量复制附件到导出目录
- 按会话 hash 组织目录结构
- 返回成功复制的文件数量

**目录结构**：
```
{outputPath}/
└── attachments/
    ├── {sessionHash1}/
    │   ├── 123456.pdf
    │   ├── 789012.docx
    │   └── ...
    ├── {sessionHash2}/
    │   └── ...
    └── ...
```

### 4. 集成到导出流程（Exporter.cpp）

在 `runImpl()` 函数中，保存导出上下文之后添加：

```cpp
// Export document attachments (PDF, Word, Excel, etc.)
if (!m_cancelled)
{
    m_logger->write(m_resManager.getLocaleString("Scanning document attachments..."));

    std::vector<AttachmentInfo> attachments;
    if (m_iTunesDb->scanDocumentAttachments(attachments))
    {
        m_logger->write(formatString(m_resManager.getLocaleString("Found %d document attachment(s)."), (int)attachments.size()));

        int copied = m_iTunesDb->copyAttachments(attachments, m_output);

        if (copied > 0)
        {
            m_logger->write(formatString(m_resManager.getLocaleString("Exported %d document attachment(s) to attachments/ directory."), copied));
        }
        else
        {
            m_logger->write(m_resManager.getLocaleString("No document attachments were exported."));
        }
    }
    else
    {
        m_logger->write(m_resManager.getLocaleString("No document attachments found."));
    }
}
```

## 🔍 代码审查

### 优点

1. **最小侵入性**
   - 只添加新功能，不修改现有逻辑
   - 利用现有的 `copyFile()` 方法
   - 利用现有的 `endsWith()` 和 `toUpper()` 工具函数

2. **健壮性**
   - 空指针检查
   - 路径验证
   - 目录自动创建

3. **可维护性**
   - 代码结构清晰
   - 注释完整
   - 易于扩展（添加新的文件类型）

### 潜在问题

1. **文件名不是原始名称**
   - 当前：123456.pdf
   - 理想：项目报告.pdf
   - 原因：需要解析消息内容才能获取原始文件名

2. **无法在 HTML 中显示**
   - 附件独立存储，未与聊天记录关联
   - 需要解析消息才能在 HTML 中生成链接

3. **大小写敏感性**
   - 使用 `toUpper()` 处理扩展名大小写
   - 应该能正确处理 .PDF, .Pdf 等
