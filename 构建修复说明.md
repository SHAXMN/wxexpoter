## 📊 技术方案

### 为什么使用符号链接?

项目文件中硬编码了旧版本的库名:
```
libimobiledevice-1.0.6.dylib  (旧)
libimobiledevice-1.0.dylib    (新)
```

通过创建符号链接,让旧名称指向新库:
```bash
ln -sf 新库路径 releases/windows-libs/x64/rel/bin/旧库名
```

**优势**:
- 不需要修改 Xcode 项目文件
- 自动适配 Homebrew 版本更新
- 保持项目结构不变

## 💡 长期改进建议

### 1. 使用 pkg-config (推荐)

自动获取正确的链接参数,不依赖硬编码的版本号:

```bash
pkg-config --libs libimobiledevice libimobiledevice-glue
```

### 2. 修改项目文件

直接编辑 `project.pbxproj`,去掉所有库名中的版本号:

```
-limobiledevice-glue-1.0.0  →  -limobiledevice-glue
-lusbmuxd-2.0.6             →  -lusbmuxd
```

### 3. 迁移到 CMake

使用 CMake 替代 Xcode 项目,更容易维护:

```cmake
find_package(PkgConfig REQUIRED)
pkg_check_modules(IMOBILEDEVICE REQUIRED libimobiledevice-1.0)
```

### 4. 清理废弃 API

继续替换其他废弃的 API:
- `NSOKButton` → `NSModalResponseOK`
- `NSClosableWindowMask` → `NSWindowStyleMaskClosable`
- `sprintf` → `snprintf`

## ⚠️ 注意事项

### dyld 警告可以忽略

构建日志中的这些警告不影响编译:
```
dyld[7142]: symbol '_kFigCaptionTextAlign_Start' missing...
```

这是 macOS 系统框架的内部问题,可以安全忽略。

### 其他警告

大量的 deprecation warnings 不会导致编译失败,可以逐步修复。
