### 2. 附件处理函数

**关键函数**：`parseAppMsgAttachment()`（来源：`MessageParser.cpp`）

```cpp
bool MessageParser::parseAppMsgAttachment(const WXAPPMSG& appMsg, 
                                         const XmlParser& xmlParser, 
                                         const Session& session, 
                                         TemplateValues& tv) const
{
    std::string title;
    std::string attachFileExtName;
    
    // 解析标题
    xmlParser.parseNodeValue("/msg/appmsg/title", title);
    
    // 解析文件扩展名
    xmlParser.parseNodeValue("/msg/appmsg/appattach/fileext", attachFileExtName);
    
    // 构建文件路径
    std::string attachFileName = m_userBase + "/OpenData/" + session.getHash() + "/" + appMsg.msg->msgId;
    std::string attachOutputFileName = appMsg.msg->msgId;
    
    // 如果有扩展名，添加到文件名
    if (!attachFileExtName.empty())
    {
        attachFileName += "." + attachFileExtName;
        attachOutputFileName += "." + attachFileExtName;
    }
    
    // 调用通用文件解析函数
    return parseFile(combinePath(m_outputPath, session.getOutputFileName()), 
                    DIR_ASSETS, DIR_ASSETS, 
                    attachFileName, attachOutputFileName, title, tv);
}
```

### 3. 通用文件处理

**关键函数**：`parseFile()`（来源：`MessageParser.cpp`）

```cpp
bool MessageParser::parseFile(const std::string& sessionPath, 
                              const std::string& sessionAssetsPath, 
                              const std::string& sessionAssetsUrlPath, 
                              const std::string& src, 
                              const std::string& dest, 
                              const std::string& fileName, 
                              TemplateValues& tv) const
{
    bool hasFile = false;
    if (!m_options.isTextMode())
    {
        // 从 iTunes 备份中复制文件到输出目录
        hasFile = m_iTunesDb.copyFile(src, combinePath(sessionPath, sessionAssetsPath), dest);
    }

    if (hasFile)
    {
        // 文件复制成功，生成下载链接
        tv.setName("plainshare");
        tv["%%SHARINGURL%%"] = sessionAssetsUrlPath + "/" + dest;
        tv["%%SHARINGTITLE%%"] = fileName;
        tv["%%MESSAGE%%"] = "";
        tv["%%MSGTYPE%%"] = "file";
    }
    else
    {
        // 文件复制失败，显示文件名
        tv.setName("msg");
        tv["%%MESSAGE%%"] = formatString(m_resManager.getLocaleString("[File: %s]"), fileName.c_str());
    }
    
    return true;
}
```

## 🔍 工作原理

### 附件导出流程

1. **识别消息类型**
   - WechatExporter 解析消息 XML
   - 识别 `APPMSGTYPE_ATTACH` (类型 6) 为文件附件

2. **提取文件信息**
   - 从 XML 中提取文件标题：`/msg/appmsg/title`
   - 从 XML 中提取文件扩展名：`/msg/appmsg/appattach/fileext`
   - 构建源文件路径：`{userBase}/OpenData/{sessionHash}/{msgId}.{ext}`

3. **复制文件**
   - 从 iTunes 备份中定位文件
   - 复制到导出目录的 `assets` 子目录
   - 保留原始文件扩展名

4. **生成 HTML 链接**
   - 在导出的 HTML 中生成下载链接
   - 用户可以直接点击下载附件

### 关键特性

- ✅ **扩展名无关**：通过 `fileext` 字段动态获取扩展名，支持任意格式
- ✅ **原始文件保留**：直接复制原始文件，不进行转换或修改
- ✅ **文件名保留**：保留原始文件名和扩展名
- ✅ **下载链接**：在 HTML 中生成可点击的下载链接

## ⚠️ 注意事项

### 1. 文件完整性

- ✅ 如果文件在 iTunes 备份中存在，会被完整复制
- ❌ 如果文件不在备份中（例如已过期或被清理），会显示 `[File: 文件名]` 但无法下载

### 2. 文件大小

- iTunes 备份可能不包含所有大文件
- 微信可能会清理旧的大文件以节省空间
- 建议及时备份重要文件

### 3. 文件类型限制

- **无限制**：WechatExporter 不对文件类型做任何限制
- 只要文件在 iTunes 备份中，就会被导出
- 文件扩展名由微信消息 XML 中的 `fileext` 字段决定

## ✅ 最终结论

**WechatExporter 支持导出所有类型的文件附件**，没有任何格式限制。

核心原因：
1. 使用通用的文件复制机制（`copyFile`）
2. 动态读取文件扩展名（`fileext`）
3. 不对文件内容进行解析或转换
4. 直接复制原始二进制文件

只要文件在 iTunes 备份中存在，WechatExporter 就能将其导出到 HTML 页面中，并提供下载链接。

- 定期进行 iTunes 备份，确保重要文件被包含
- 导出后检查 `assets` 目录，确认重要附件已成功导出
- 对于特别重要的文件，建议单独备份
